# Using Symmetry Definitions in Morphing (Ansys Meshing)

This guide explains how to apply **Symmetry Definitions** when performing **Direct Morphing** in Ansys Meshing. Symmetry definitions ensure that, while nodes are displaced by the morphing operation, the geometric symmetry of the model (planar, cyclic, or periodic) is preserved. This is essential when working with rotationally periodic sectors (turbomachinery blades, electric motor sectors, pump impellers) or when shape changes must remain mirrored across a plane.

---

## 1. When to Use Symmetry in Morphing

Use a symmetry definition in a morphing step whenever:

- The source mesh was built on a **periodic sector** (e.g., a 1/N slice of a full 360° model) and the morph must not break node pairing on the periodic faces.
- The model has a **mirror-symmetry plane** and the deformation must stay symmetric about that plane.
- You intend to **reassemble a full model by rotational copy** after morphing, and the periodic interfaces must remain node-to-node conformal.
- A downstream solver (CFD, structural, electromagnetic) expects matching meshes on the periodic or symmetric boundaries.

Without a symmetry definition, a direct morph will displace nodes independently on each side of the periodic boundary, destroying node matching and invalidating periodic boundary conditions.

---

## 2. Prerequisites: Define Labels for the Scope

Before the symmetry control can be added to a morphing operation, you must **define Named Selections (labels)** that identify the geometric entities involved. The morphing operation and the symmetry control both reference these labels to determine their scope.

Create the following Named Selections in the Meshing tree (right-click **Named Selections → Insert → Named Selection**):

| Label (Named Selection) | Scope | Purpose |
|---|---|---|
| `morph_target` | Faces or edges that will move | The geometry to be deformed by the direct morph |
| `morph_fixed` | Faces or edges held fixed | Nodes that must not move during morphing |
| `sym_side_A` | Source periodic face | First side of the periodic pair |
| `sym_side_B` | Target periodic face | Second side of the periodic pair (must match `sym_side_A` topology) |
| `sym_axis` *(periodic only)* | Axis of revolution | Required for rotational/cyclic periodicity |
| `sym_plane` *(planar only)* | Mirror plane | Required for planar (reflective) symmetry |

**Important.** The two periodic faces (`sym_side_A` and `sym_side_B`) must have a consistent node pattern — either because they were generated by a match-control / cyclic mesh, or because the mesh was built on a single sector and rotationally copied. The symmetry definition in the morph preserves this pairing; it does not create it.

---

## 3. Adding the Periodic Symmetry Control

The symmetry control is added as a child of the Direct Morphing step so that it is active during that operation.

### Steps

1. In the Meshing outline, locate (or insert) the **Direct Morphing** operation under the Mesh branch.
2. Right-click the Direct Morphing step and choose **Insert → Symmetry Definition** (or, depending on the release, **Insert → Periodic Symmetry Control**).
3. In the **Details** view of the new symmetry control, configure:
   - **Type:** `Periodic` for rotational/cyclic, or `Planar` for mirror symmetry.
   - **Scope — Side A:** select the Named Selection `sym_side_A`.
   - **Scope — Side B:** select the Named Selection `sym_side_B`.
   - **Axis Definition** (periodic only): reference the `sym_axis` Named Selection, or specify the axis by coordinate system and direction.
   - **Sector Angle** (periodic only): angle of the periodic sector, e.g. `30°` for a 12-bladed machine.
   - **Symmetry Plane** (planar only): reference the `sym_plane` Named Selection or a coordinate-system plane.
4. Confirm that the Direct Morphing step itself is scoped to `morph_target` (nodes to move) and, if needed, `morph_fixed` (nodes to hold).

You may add **more than one** symmetry definition to a single morphing step — for example, one periodic definition for a cyclic sector plus a planar definition for a hub mid-plane. Each is evaluated during the morph and applied to its scoped faces.

---

## 4. Behavior During Morphing

With the symmetry definition active, the morpher enforces the following during the displacement solve:

- For a **periodic** control: any nodal displacement applied to a node on `sym_side_A` is reproduced on its paired node on `sym_side_B` after rotation through the sector angle about the axis. Node pairing is preserved.
- For a **planar** control: displacements normal to the mirror plane are mirrored, and in-plane components are duplicated on the paired node on the opposite side.
- Nodes scoped by `morph_fixed` remain at their original coordinates regardless of the morph.
- Nodes in the interior of the domain are smoothed consistently with the boundary constraints so that element quality on the periodic interface is preserved.

---

## 5. Validation After Morphing

After running the morph, verify the result before handing the mesh to the solver:

1. **Check periodic node matching.** Use a periodicity or mesh-match check: the paired node distance after rotation should be within tolerance (typically `1e-6` of the model characteristic length).
2. **Inspect element quality** on and near the periodic faces (skewness, orthogonal quality). A broken symmetry definition usually shows up as degraded elements along the interface.
3. **Visual check.** Rotationally copy the sector for display; the periodic seam should be invisible.
4. **Solver-side check.** When the mesh is imported, the downstream solver's periodic or cyclic-symmetry boundary condition should accept the interface without a mismatch warning.

---

## 6. Common Pitfalls

- **Scoping the morph to the whole body instead of `morph_target`.** The morph then tries to move periodic faces freely; even with a symmetry control, results can be unstable. Keep the morph scope tight.
- **Non-matching Side A and Side B meshes.** The symmetry control assumes a pairing already exists. If the two faces were meshed independently and do not match, create a cyclic match control in the mesh *before* morphing.
- **Wrong axis or sector angle.** A periodic control with the wrong axis silently produces a deformed but asymmetric mesh. Verify the axis Named Selection before running.
- **Mixing planar and periodic scopes on the same faces.** Each face should belong to at most one symmetry definition.
- **Forgetting to update the Named Selections** after a CAD change. The label still exists but scopes different geometry, so the symmetry control applies to the wrong faces.

---

## 7. Quick Reference

```
Mesh
└── Direct Morphing (scope: morph_target, fixed: morph_fixed)
    ├── Symmetry Definition — Periodic
    │   ├── Side A: sym_side_A
    │   ├── Side B: sym_side_B
    │   ├── Axis:   sym_axis
    │   └── Sector Angle: <value>
    └── Symmetry Definition — Planar (optional, additional)
        └── Plane: sym_plane
```

Define the labels first, attach the symmetry definition to the morphing step, and validate node pairing after the morph runs.
